byo·claw

Bring your own OpenClaw into third party websites.

A specification for builders who want to add byo·claw integration to their products.

Specification Version

The byo·claw specification is currently at version 0.1.0

Specification Overview

byo·claw is fundamentally 2 pieces:

  • A restricted, separate api for OpenClaws
  • A prompt to a human’s Claw

Important Caveats

  • Claw tokens are short-lived (we suggest 10 minutes, never more than 60 minutes)
  • Claws cannot do destructive actions
  • Claws cannot do dangerous actions
  • A claw token can only be obtained by exchanging a human-instigated login token (ie. a website cookie)
  • We have a specification for prompts that should be adhered to so a human can scan it and see it is mundane

Additional Features

  • Claw tokens can be renewed †
  • Capability tokens are bound to a single API base path via api_base (for example /api/claw).
  • Claws may request renewal via a hosted URL derived from old-token context, and renewal succeeds only for an active logged-in user session.
† The claw is given a URL specification and a puzzle to create the URL based on their previous token. The claw can then present this URL to their human. Clicking it opens a page at the original site that allows the user to renew the token provided they are signed in as the same user who created the original token.

Specification Development

The specification is a living, open source document. Participation from both humans and machines is encouraged.

Please have your Claw check the issue tracker for existing issues before opening a new one.

Other Contributions

SDKs are welcome in this repo. Any language is fine. We already have js/ts via npm i byoclaw.

AI contributions are accepted, but poor quality submissions may be banned.
